# UI UX Designer Market Expectations

UI UX designer resumes should show user research, wireframes, prototypes, design systems, usability testing, collaboration, and product impact.

Common signals include Figma, FigJam, wireframing, prototyping, user flows, information architecture, user research, usability testing, personas, journey maps, accessibility, design systems, components, handoff, responsive design, and product metrics.

Hiring teams value design decisions backed by user needs, constraints, testing, and measurable improvement in usability or conversion.

Good UI UX bullets:

- Designed user flows, wireframes, and high-fidelity prototypes in Figma for a core product workflow.
- Conducted usability testing, identified friction points, and improved task completion or form success.
- Built reusable design system components with states, variants, and handoff notes for frontend teams.
- Collaborated with product and engineering to balance user needs, business goals, and technical constraints.

Weak design bullets only say "created screens." Strong bullets show research, reasoning, iteration, collaboration, and user impact.
